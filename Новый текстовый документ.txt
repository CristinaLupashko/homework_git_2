1 уровень сложности: 1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
1.Чтобы отказаться от изменений в файле, находящемся в рабочем каталоге, можно использовать команду git checkout, которая возвращает файл к его состоянию в последнем коммите.

2.Как вернуть файл из индекса в рабочий каталог?
2.Чтобы вернуть файл из индекса в рабочий каталог, можно использовать команду git reset.
Если вы хотите вернуть конкретный файл, выполните следующую команду:
git reset имя_файла
После выполнения этой команды изменения, связанные с файлом, будут удалены из индекса, и файл вернется к своему последнему состоянию в рабочем каталоге.

3.Как отменить ошибочный commit ?
3.Если вы случайно сделали неправильный коммит, вы можете отменить его с помощью команды git reset.
Если вы хотите удалить последний коммит, но сохранить все изменения в рабочем каталоге, выполните следующую команду:
git reset HEAD~1
Если вы хотите удалить последний коммит и изменения в рабочем каталоге, выполните следующую команду:
git reset --hard HEAD~1
Эта команда удаляет последний коммит и откатывает все изменения в рабочем каталоге к состоянию на момент предыдущего коммита.

4.Как создать новую ветку и переключится в нее?
4.Чтобы создать новую ветку и переключиться в нее, выполните следующие команды:
git branch имя_новой_ветки
git checkout имя_новой_ветки
ервая команда git branch создает новую ветку с указанным именем, но не переключается в нее. Вторая команда git checkout переключает ветку на новую ветку.
Кроме того, вы можете выполнить эти две команды одновременно с помощью команды git checkout с флагом -b:
git checkout -b имя_новой_ветки.

5.Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?
Чтобы создать новый Git репозиторий из локального каталога на своем компьютере и отправить его на Github, выполните следующие шаги:

Перейдите в каталог, который вы хотите сделать Git репозиторием.
Инициализируйте репозиторий с помощью команды git init. Эта команда создаст скрытую папку .git, которая будет хранить историю изменений и другую информацию о репозитории.
Создайте файлы и директории, которые вы хотите добавить в репозиторий.
Добавьте файлы в индекс с помощью команды git add. Например, если вы хотите добавить все файлы в текущем каталоге, выполните команду git add ..
Сделайте коммит изменений с помощью команды git commit. Вы можете использовать флаг -m для добавления сообщения к коммиту. Например, git commit -m "Initial commit".
Создайте новый репозиторий на Github.com и скопируйте его URL.
Связать локальный репозиторий с удаленным репозиторием, используя команду git remote add origin <URL репозитория>. Замените <URL репозитория> на URL вашего нового репозитория на Github.
Отправьте свой локальный репозиторий на Github с помощью команды git push -u origin master. Эта команда отправляет изменения из ветки master вашего локального репозитория в ветку master на Github.
Важно заметить, что при отправке вашего репозитория на Github, вы должны быть авторизованы на Github и иметь разрешение на создание репозиториев в вашей учетной записи Github.
